Available since v1.7.0, undocumented

The purpose of Adapter.relax method is to make sure that the uiScroll relaxes and there are no internal processes working at the moment we need to run some logic. Basically, it may be treated as a shortcut for the following approach, which is presented on "Is loading" tab: 

{{isLoadingSample}}

The Adapter.relax method accepts 1 optional argument, it is a callback that allow to run necessary logic right before resolving "relax" promise. This might be helpful if we don't want to wait until the end of the call stack. The "Relax cb" tab shows the difference between "await" and "callback" approaches.

This sample is quite artificial, because for 2 manual operations (remove and insert) the second one can be done safely by using Return value API, which is a part of each Adapter method. This approach is demonstrated on the last tab "Return".

But the point is that the first manual operation also might require safe call. If we have a chance that there are some pending processes at the moment we are going to run the Adapter method, then we should wait until these processes are done. The "Is loading" approach is about it. But having the "Relax" shortcut we may make our life easier.
