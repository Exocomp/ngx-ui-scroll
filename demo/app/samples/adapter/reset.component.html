Adapter.reset method is designed to reset the internal state of the uiScroll. It differs from Adapter.reload because it re-instantiates all internal entities, which is equivalent to destroying the scroller and re-creating it again via ngIf.

The method has 1 optional argument: datasource. By passing it, we tell the uiScroll that we want to use a new datasource. If the argument is not passed, the old datasource will be used after re-initialization.

In this demo the Adapter.reset method is used to change startIndex and bufferSize settings while Datasource.get remains pristine. It affects the value of the DOM elements counter and the size of the scrollable area.
